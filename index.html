Understanding the Problem

What is an internet blackout?

The internet isn't one thing. It's thousands of separate networks all agreeing to pass traffic to each other. Your home WiFi is a network. Your mobile carrier is a network. Google runs networks. So does every country.

When you visit a website hosted in another country, your request hops through multiple networks to get there. Each network knows which neighbors it can reach, and they pass your data along like a chain of messengers.

Think of it this way

The internet is like a city of buildings connected by roads. Each building (network) has its own rooms and people. The roads let traffic flow between buildings. Normally, if one road closes, you can take another route.

Most countries have multiple connections to the global internet through different providers in different locations. This redundancy makes the internet resilient. If one path fails, traffic finds another way.

In Iran, this redundancy has been systematically eliminated. All international traffic now flows through one single point—making a nationwide blackout trivially easy to execute.

The Vulnerability

A single point of failure for 87 million people

All of Iran's international internet traffic passes through one gateway: AS49666, operated by the state-controlled Telecommunication Infrastructure Company (TIC). Every mobile user, every home connection, every business—all dependent on this single point.

Why this matters

With only one gateway, disconnecting an entire nation requires just one action. There's no redundancy, no backup path, no resilience. When this gateway stops announcing routes, Iran disappears from the internet.

The Technical Reality

How a blackout actually happens

The internet uses a protocol called BGP (Border Gateway Protocol) to know how to route traffic. Think of BGP as a global address book that every network constantly updates. When a network wants to be reachable, it "announces" its addresses to its neighbors.

During a blackout, Iranian networks stop announcing their addresses. The rest of the world's routers gradually forget how to reach Iran. It's not blocking—it's erasure.

Think of it this way

Imagine a country being erased from every GPS and map in the world simultaneously. People wouldn't refuse to go there—they simply couldn't find it. That's what BGP withdrawal does.

How We Got Here

The systematic elimination of redundancy

Iran didn't always have this vulnerability. Over a decade, the number of international gateways was deliberately reduced from five to one, concentrating all control in state hands.

The "National Information Network" (NIN), launched in 2017, was marketed as improving domestic internet speed. In reality, it created a two-tier system: domestic services continue during blackouts while international access is severed.

DNS Tunneling
Technical / Slow

Encodes data inside DNS queries. DNS is essential for basic internet function, so it's harder to block completely.

How it works
Data is encoded into fake DNS lookups. A server outside Iran decodes these and fetches real content. Tools: iodine, dnscat2.
Limitation
Extremely slow (<50 kbps). Only useful for text.
üßÖ
Tor with Pluggable Transports
Moderate difficulty

Tor's "pluggable transports" (obfs4, meek, Snowflake) disguise traffic to look like normal HTTPS or video calls.

How it works
Connects to "bridges" that obfuscate traffic. Snowflake uses WebRTC (video call protocol) to tunnel through.
Limitation
Bridge IPs get blocked. Connection unreliable. Requires setup.

Satellite Internet (Starlink)
High risk / Expensive

Satellite connections bypass terrestrial infrastructure entirely. Starlink has activated service over Iran.

How it works
Dishes communicate directly with satellites, skipping ground-based chokepoints entirely.
Limitation
Equipment must be smuggled. Dishes detectable. Possession illegal. Iran uses jamming.

Mesh Networks (Briar, Bridgefy)
Requires nearby users

Phone-to-phone communication via Bluetooth or WiFi, without internet. Messages hop between devices.

How it works
Apps create encrypted local networks. Messages pass through nearby phones until reaching recipients.
Limitation
Only local communication. Requires many users nearby. Can't access global internet.

Psiphon / Lantern
User friendly

Purpose-built circumvention tools that automatically try multiple bypass techniques and rotate servers.

How it works
Combines VPN, SSH tunneling, and obfuscation. Maintains thousands of servers, auto-switches when blocked.
Limitation
Downloads blocked. Must get APK through unofficial channels. Speed varies.

Domain Fronting
Mostly blocked

Makes traffic appear to go to allowed services (like Google) by exploiting CDN routing.

How it works
Outer connection shows "google.com" but encrypted header requests blocked site. CDN routes accordingly.
Limitation
Google, Amazon disabled this in 2018. Few viable domains remain.
Preparation is key

No single method is reliable. The best approach is to have multiple tools ready before a blackout occurs. Download Tor, Psiphon, and mesh apps in advance. Share via Bluetooth when app stores are blocked.
